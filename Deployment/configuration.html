Deployment Steps

Authenticate to Target Environment

Connect the PAC CLI to your Dataverse environment.

$ pac auth create --environment "https://your-org.crm3.dynamics.com"

Import the Schema Solution

This creates all 16 tables, columns, and relationships required by the flows.

$ pac solution import \
    --path nintex_1_0_0_1_managed.zip \
    --publish-changes \
    --activate-plugins

Import the Workflow Solution

Deploys the 10 Power Automate cloud flows and Dataverse connection reference.

$ pac solution import \
    --path ESignatureBroker_1_0_0_33_managed.zip \
    --publish-changes \
    --activate-plugins

Verify Import

Confirm both solutions appear in the environment.

$ pac solution list

# Expected output:
# nintex              1.0.0.1    Managed
# ESignatureBroker    1.0.0.33   Managed

Configure & Activate

See the Configuration tab for post-deployment setup steps.

📦

Alternative: Power Platform Admin Center

Import Workflow Solution

Post-Deployment Configuration

After importing both solutions, complete these configuration steps.

🔗

1. Map Connection Reference

The cs_sharecommondataserviceforapps connection reference must be linked to a Dataverse connection in the target environment.

Open Connection References

Solutions → E-Signature Broker → Connection References → Dataverse (Current Environment)

Select Connection

Click Edit, then choose an existing Dataverse connection or create a new one using the service account that should run the flows.

Save

Save the connection reference. This unlocks all 10 flows for activation.

⚡

2. Activate Cloud Flows

All flows import in a draft (off) state. Activate them after configuring the connection reference.

ℹ
Navigate to Solutions → E-Signature Broker → Cloud flows. Select each flow and click Turn on.
🔑

3. Configure Nintex eSign Credentials

Create a configuration record in the cs_assuresign table with your Nintex API credentials.

ColumnValueDescription
cs_apiurlYour Nintex eSign API URLe.g., https://api.assuresign.net/api
cs_apikeyYour API keyFrom Nintex admin portal
cs_accountidYour account IDGUID from Nintex account settings
cs_nameConfiguration namee.g., "Production Config"
🔒
Security Note. The API key provides full access to your Nintex eSign account. Restrict access to the cs_assuresign table using Dataverse security roles.
🔨

4. Verify Integration

Create a Test Envelope

Create a new record in the cs_envelope table with the required fields populated.

Check Flow Execution

The ESign - Prepare Envelope flow should trigger automatically. Check flow run history for success.

Test Send

Set cs_issent to Yes on the envelope record to trigger the send flow.

Cloud Flows Reference

All 10 Power Automate flows and their trigger conditions.

ESign - Prepare Envelope
Trigger: cs_envelope record created
Initializes a new envelope in Nintex, creates associated records (documents, signers, fields), and sets the envelope ID.
ESign - Send Envelope
Trigger: cs_issent changed to Yes
Submits the prepared envelope to Nintex for signing. Updates status and retrieves the Nintex envelope ID.
ESign - Status Sync
Trigger: cs_requeststatus changed to Yes
Polls Nintex for the current envelope status and updates all related Dataverse records (signers, documents).
ESign - Get Signing Links
Trigger: cs_signer.cs_requestlink changed to Yes
Retrieves the unique signing URL for a specific signer from Nintex and stores it on the signer record.
ESign - Get Access Links
Trigger: cs_accesslink record created
Fetches access link URLs for envelope viewing (non-signer access).
ESign - Get Document Content
Trigger: cs_document.cs_requestsignedcopy changed to Yes
Downloads the signed PDF document from Nintex and stores the content in Dataverse.
ESign - Get Envelope History
Trigger: cs_envelope.cs_requesthistory changed to Yes
Retrieves the full audit history from Nintex and creates corresponding history records in Dataverse.
ESign - Cancel Envelope
Trigger: cs_envelope.cs_iscancelled changed to Yes
Sends a cancellation request to Nintex and updates the envelope status in Dataverse.
ESign - Send Signer Reminder
Trigger: cs_signer.cs_sendreminder changed to Yes
Sends a reminder notification to a signer who hasn't yet completed signing.
ESign - Sync Templates
Trigger: cs_assuresign.cs_synctemplates changed to Yes
Imports available signing templates from the Nintex account into the Dataverse template table.

Troubleshooting

Common issues and their resolutions.

⚠

Import Errors

ErrorCauseResolution
NullReferenceException (0x80040216) Workflow XML format uses attributes instead of child elements Use the provided managed solution zips — they have the correct format
WorkflowOperationParametersExtraParameter Missing columns in schema (flow references a column that doesn't exist) Ensure the schema solution (nintex) is imported first
Attribute is a String, but Lookup specified (0x80041A06) Attempting to change an existing column type This is resolved — flows use cs_envelopelookup (new column) instead of modifying cs_envelopeid
Connection reference not found Connection reference not yet mapped Edit the connection reference and select a Dataverse connection
Flow won't turn on Connection reference not configured Configure the connection reference first, then activate flows

Uninstalling

⚠
Managed solutions delete data. Removing a managed solution will delete all tables, columns, and data defined by that solution. Export any required data before uninstalling.
# Remove workflow solution first (depends on schema)
$ pac solution delete --solution-name ESignatureBroker

# Then remove schema solution
$ pac solution delete --solution-name nintex
